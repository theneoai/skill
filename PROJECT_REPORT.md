## 三、架构设计

### 核心+适配器模式

采用"核心引擎 + 平台适配器"的设计模式：
- **核心引擎**：平台无关的 YAML 配置定义
- **适配器层**：各平台的 JavaScript 实现
- **构建工具**：将核心内容嵌入平台模板

### 三大操作模式

#### 1. CREATE 模式（7步工作流）
```
解析请求 → 选择模板 → 引导需求（6个问题）→ 生成输出 → 安全扫描 → 质量检查 → 交付
```
支持4种模板：基础、API集成、数据管道、工作流自动化

#### 2. EVALUATE 模式（4阶段管道，满分1000分）

| 阶段 | 分值 | 内容 |
|------|------|------|
| Phase 1: 解析与验证 | 100分 | YAML语法、结构、元数据 |
| Phase 2: 文本质量 | 300分 | 清晰度、完整性、准确性、安全性、可维护性、可用性 |
| Phase 3: 运行时测试 | 400分 | 单元、集成、沙箱、错误处理、性能、安全测试 |
| Phase 4: 认证 | 200分 | 文档、测试覆盖率、代码质量、兼容性、审查 |

**认证等级：**
- BRONZE: 600-699分
- SILVER: 700-799分
- GOLD: 800-899分
- PLATINUM: 900-1000分

#### 3. OPTIMIZE 模式（9步优化循环）

7个分析维度：简洁性、清晰度、完整性、安全性、性能、可维护性、可用性

收敛检测：当改善低于5分或连续3次无增益时停止优化。
