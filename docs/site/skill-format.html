SKILL.md Format

YAML-based skill specification format for AI skill lifecycle management. Each skill is defined by structured YAML frontmatter followed by markdown sections.

📋 YAML Frontmatter

Required metadata block at the top of every SKILL.md file, delimited by --- lines.

name string Unique skill identifier in kebab-case required
version string Semantic version (e.g., "1.0.0", "2.15.0") required
description map Bilingual description: {en: string, zh: string} required
license string SPDX identifier (e.g., MIT, Apache-2.0)
author map {name: string, email?: string}
created string ISO 8601 date (YYYY-MM-DD)
updated string ISO 8601 date (YYYY-MM-DD)
tags list Universal taxonomy for classification
interface map {input, output, modes[]}
extends map Implementation-specific extensions (evaluation, security, evolution)
Tags Taxonomy: lifecycle, quality-assurance, security, autonomous, multi-agent, evaluation, error-recovery

🏷️ Mode Interface

Skills declare supported modes via the interface.modes field:

CREATE
EVALUATE
LEAN
RESTORE
SECURITY
OPTIMIZE

🔧 extends — Implementation Extensions

Tool-specific configuration for evaluation, security scanning, and evolution.

evaluation

# Example
evaluation:
  metrics: [f1, mrr]
  thresholds:
    f1: 0.90
    mrr: 0.85

security

security:
  standard: CWE
  scan-on-delivery: true

📄 Markdown Sections

After the frontmatter, content is organized into numbered sections (§1.1, §1.2, etc.):

§1.1 Identity Name, role, purpose, design patterns, core principles, red lines
§1.2 Framework Architecture overview and mode routing
§1.3 LOOP Plan-Execute-Summarize phases and exit conditions
§2.0 Multi-LLM Deliberation LLM roles, consensus protocol, error recovery
§3.0 Inversion Pattern Requirements elicitation methodology
§4.0 Audit Trail Required fields and storage specification
§5.0 Security Red Lines CWE violations and ABORT protocol
§6.0 Usage Examples Example inputs/outputs for each mode

✨ Example: skill Skill

The actual SKILL.md for the skill framework itself:

skill/SKILL.md 
# Universal Skill Metadata
name: skill
version: 2.15.0
description:
  en: "Full lifecycle AI skill engineering system..."
  zh: "全生命周期AI技能工程系统..."

license: MIT
author:
  name: theneoai
  email: lucas_hsueh@hotmail.com
created: 2024-01-15
updated: 2026-03-30

tags:
  - lifecycle
  - quality-assurance
  - security
  - autonomous
  - multi-agent
  - evaluation
  - error-recovery

interface:
  input: user-natural-language
  output: structured-skill
  modes: [create, evaluate, restore, security, optimize]

extends:
  evaluation:
    metrics: [f1, mrr]
    thresholds: {f1: 0.90, mrr: 0.85}
  security:
    standard: CWE
    scan-on-delivery: true
---

📏 Quality Thresholds

Skills must meet these measurable quality gates for certification:

evaluation:
  metrics: [f1, mrr]
  thresholds:
    f1: 0.90   # Minimum 90% F1 score
    mrr: 0.85   # Minimum 85% MRR
